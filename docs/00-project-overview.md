# 项目概览

## 1. 项目简介

本项目是一个 AI 聊天式电商系统（AI Chat E-commerce Demo），目标是在单个工程中完成“内容流浏览 + AI 导购 + 下单支付”的演示型闭环。

系统当前聚焦于 PC Web 场景，核心业务围绕以下能力展开：

- AI 聊天搜索商品
- 规则驱动的 AI 商品推荐与动作返回
- 用户注册、登录、JWT 鉴权
- 商品列表与关键词搜索
- 真实购物车
- 简化订单系统
- 支付宝 PC 跳转支付
- 订单查询与支付状态展示

说明：

- 当前代码仓库中，AI 聊天能力已集成在后端 `chat` 模块中。
- DeepSeek、商品 embedding 语义检索仍属于规划能力，`chat_session_context` 与基础引用解析已在当前代码中落地。

---

## 2. 当前项目目标

当前阶段目标是实现一个具备主链路能力的 AI 电商 Demo，覆盖以下业务路径：

用户登录
-> 浏览商品
-> AI 搜索/推荐商品
-> 加入购物车 / 立即购买
-> 创建订单
-> 发起支付宝 PC 支付
-> 查看订单

当前目标强调：

- 前后端分离开发
- 真实后端业务接口
- 简化但可演示的交易链路
- AI 返回结构可被前端稳定渲染

---

## 3. 当前系统架构

当前系统按职责可分为以下模块：

- `frontend`
  当前代码目录为 `frontend`，负责登录、注册、首页、聊天页、购物车、订单页、支付结果页，以及对后端 API 的调用与页面渲染。
- `mall-backend`
  当前代码目录为 `backend/mall-backend`，负责用户认证、商品、购物车、订单、支付、AI 聊天编排等核心业务，是唯一业务事实来源。
- `ai-module`
  当前仓库中未单独落地为独立目录；AI 能力实际并入 `mall-backend` 的 `chat` 模块。独立 AI 模块仍停留在架构规划层。
- `crawler`
  当前仓库中未落地为独立实现目录，属于后续商品采集与数据准备规划。
- `monitor`
  当前仓库中未落地为独立实现目录，监控能力仍停留在规划层。
- `docs`
  存放项目范围、架构、数据库、API、任务板、联调规则和 UI/UX 说明，是当前阶段的冻结依据。
- `docker`
  当前已提供 PostgreSQL 相关初始化脚本，包含 pgvector 扩展开启脚本。
